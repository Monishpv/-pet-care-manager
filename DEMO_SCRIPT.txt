[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's walk through the system architecture.
Our entry point is the START node, which routes directly to our custom Security Checkpoint to analyze incoming queries.
If the request passes the security checks, it flows to the central PetCare Orchestrator.
The Orchestrator coordinates two specialized sub-agents: the Diet Scheduler, which calculates portions, and the Health Tracker, which reviews clinical data.
These sub-agents interact directly with our Model Context Protocol server.
Finally, if any database modification is proposed, the flow routes through a Human Confirmation pause before saving, while security alerts route to a safe terminal node.
